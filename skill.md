이 파일 하나로 모든 작업이 가능하다. 별도 스킬 파일을 찾지 말 것 — 이 문서 하단의 "PART B"에 스킬 전문이 통째로 포함되어 있다.

## 트리거

사용자가 "오늘치 경력기술서 생성해줘" 또는 "오늘치 생성" 또는 "daily" 라고 입력하면 PART A의 절차를 그대로 실행한다.

---

## 사전 준비물

레포 루트에 다음만 있으면 된다:
- `CLAUDE.md` (이 파일 하나)
- `history.json` (없으면 자동 생성)
- `outputs/` (없으면 자동 생성)

# PART A: 자동화 실행 절차

### STEP 1: 본 문서의 PART B(스킬 전문) 숙지

이 파일 하단 "PART B: 경력기술서 작성 스킬 전문" 섹션의 8가지 원칙(최채용솔로가정부), 양식 A~D, 직무별 성과 표현 레퍼런스(섹션 8), 직무별 전략 패턴(섹션 8-1), docx 생성 규칙(섹션 10), 최종 점검 체크리스트(섹션 11)를 모두 적용한다.

### STEP 2: history.json 읽기

레포 루트의 `history.json`을 읽는다. 파일이 없으면 아래 초기값으로 생성한다.

```json
{
  "current_cycle": 1,
  "remaining_in_cycle": ["영업", "마케팅", "재무회계", "인사", "생산관리", "IT개발", "MD바잉", "제약영업", "품질관리", "물류SCM", "홍보"],
  "used_combos": [],
  "history": []
}
```

### STEP 3: 오늘 날짜로 이미 생성한 적 있는지 확인

`history` 배열에서 오늘 날짜(YYYY-MM-DD) 항목이 이미 있으면, 사용자에게 "오늘치는 이미 생성되었습니다. 다시 생성할까요?"라고 묻고 답을 기다린다. "응" 또는 "yes"면 해당 날짜 항목을 history에서 삭제하고 진행. 아니면 종료.

### STEP 4: 직무 3개 뽑기 (사이클 균등 분배)

- `remaining_in_cycle` 배열의 길이가 3 미만이면:
  - `current_cycle`을 1 증가
  - `remaining_in_cycle`을 11개 직무 전체로 리셋
- `remaining_in_cycle`에서 무작위로 3개 직무 선택
- 선택된 3개 직무를 `remaining_in_cycle`에서 제거

### STEP 5: 각 직무별 (연차 + 이직 상황) 조합 결정

각 직무에 대해:

**연차 풀**: 6개월~1년차, 1~3년차, 4~6년차
**이직 상황 풀**: 같은직무이동, 같은직무+업계이동, 직무변동, 사기업→공기업, 공기업→사기업

두 축을 곱하면 직무당 15개 조합. 그 중 `used_combos`에 `"{직무}|{연차}|{상황}"` 형태로 이미 존재하지 않는 조합만 후보로 두고 무작위 선택.

15개 조합이 모두 used 상태인 직무가 있으면(약 55일 이후 발생 가능), 해당 직무에 한해 used_combos에서 그 직무 항목들을 모두 제거하고 다시 추첨.

선택된 조합은 `used_combos`에 즉시 추가.

### STEP 6: 각 직무별 출력 양식 2개 결정

**양식 풀** (PART B 섹션 5와 성과 기술 변형 부분 참고):
- F1: 업무단위·간단형 (PART B 섹션 7의 패턴 1, 짧은 버전)
- F2: 업무단위·상세형 (PART B 섹션 5의 양식 A, 카테고리화 + HOW 풀어쓰기)
- F3: 프로젝트 단위형 (PART B 섹션 5의 양식 B)
- F4: 성과 강조 역순형 (PART B 섹션 5의 양식 C, 성과→HOW)
- F5: AR/TAR 한 줄 요약형 (PART B 섹션 4의 성과 기술 변형)
- F6: 인턴/저연차 정성형 (PART B 섹션 5의 양식 D)

**직무 분류:**
- 루틴직무: 영업, 재무회계, 인사, 생산관리, MD바잉, 품질관리, 물류SCM
- 프로젝트직무: 마케팅, IT개발, 제약영업, 홍보

**후보 양식 매핑 규칙:**

| 양식 | 적합 직무 | 적합 연차 |
|---|---|---|
| F1 | 모든 직무 | 모든 연차 |
| F2 | 루틴직무만 | 1~3년차, 4~6년차 |
| F3 | 프로젝트직무만 | 1~3년차, 4~6년차 |
| F4 | 모든 직무 | 1~3년차, 4~6년차 |
| F5 | 모든 직무 | 모든 연차 |
| F6 | 모든 직무 | **6개월~1년차 전용** |

직무·연차에 맞는 후보 양식 풀에서 2개를 무작위 선택. 단, 직전 5개 history 항목에서 같은 (직무·연차) 페어로 등장한 적 있는 양식 페어 조합과 동일한 페어는 회피(가능한 경우만, 후보가 부족하면 무시).

### STEP 7: 가상 인물 1명 설정

각 직무에 대해 가상 인물 1명을 만들고, 그 인물의 동일한 경력 정보를 양식 A·B 두 가지로 재구성한다.

**가상 인물 생성 규칙:**
- 이름: 한글 이름 (예: 김민수, 이서연 등) — 매번 다르게
- 회사명: 실제 기업명 사용 금지. "○○패션", "OO식품", "△△제약" 등 마스킹 표기
- 학력: "OO대학교"로 통일
- 근무기간: 연차에 맞게 역산하여 산출 (오늘 날짜 기준)
- 업무 내용·성과: PART B 섹션 8(직무별 성과 표현 레퍼런스)과 섹션 8-1(직무별 전략 패턴)을 활용해 현실적으로 작성
- 이직 상황에 따라 이전 직장과 지원 직장을 다르게 설정 (예: "사기업→공기업"이면 이전 회사는 사기업, 지원처는 공기업)

**양식 A → 양식 B 재구성 시 주의:**
- 직무, 연차, 회사명, 근무기간, 핵심 업무, 핵심 성과 수치는 동일하게 유지
- 양식 구조와 표현 방식만 다르게 변형
- 예: F2(업무단위 상세형)에서 카테고리별로 풀어 쓴 내용을 → F4(성과 강조 역순형)에서는 성과 먼저 제시 후 HOW 하위 항목으로 재배치

### STEP 8: docx 6개 생성

PART B의 모든 규칙을 준수하여 직무 3개 × 양식 2개 = 6개 docx를 생성:
- 섹션 2: 8가지 핵심 원칙(최채용솔로가정부) 전부 적용
- 섹션 3: 필수 기재 항목 7개
- 섹션 4: HOW + Result 성과 기술 공식, 임팩트 성과 1~2건 포함
- 섹션 5: 양식별 작성 가이드 (F1~F6 매핑)
- 섹션 8, 8-1: 직무별 성과 표현 레퍼런스와 전략 패턴
- 섹션 10: docx 생성 규칙 (폰트, 크기, 여백, 기호 표기 규칙 등)
- 섹션 11: 최종 점검 체크리스트

### STEP 9: 파일 저장

저장 경로: `outputs/{오늘날짜}/경력기술서_{직무}_{양식코드}.docx`

예시:
```
outputs/2026-04-29/경력기술서_마케팅_F3.docx
outputs/2026-04-29/경력기술서_마케팅_F4.docx
outputs/2026-04-29/경력기술서_재무회계_F2.docx
outputs/2026-04-29/경력기술서_재무회계_F5.docx
outputs/2026-04-29/경력기술서_생산관리_F1.docx
outputs/2026-04-29/경력기술서_생산관리_F6.docx
```

### STEP 10: history.json 업데이트

오늘 작업 결과를 `history` 배열에 append:

```json
{
  "date": "2026-04-29",
  "cycle": 1,
  "outputs": [
    {
      "job": "마케팅",
      "year": "1~3년차",
      "situation": "같은직무+업계이동",
      "formats": ["F3", "F4"],
      "person_name": "김민수",
      "files": [
        "outputs/2026-04-29/경력기술서_마케팅_F3.docx",
        "outputs/2026-04-29/경력기술서_마케팅_F4.docx"
      ]
    }
  ]
}
```

`current_cycle`, `remaining_in_cycle`, `used_combos`도 함께 업데이트.

### STEP 11: 깃허브 커밋 & 푸시

레포 루트에서 다음 명령 실행:

```bash
git add outputs/{오늘날짜}/ history.json
git commit -m "daily: {오늘날짜} 경력기술서 6편 생성 (cycle {N})"
git push
```

푸시 실패 시 사용자에게 에러 메시지 그대로 전달 (인증 문제일 수 있음).

### STEP 12: 결과 보고

사용자에게 다음 형식으로 요약 보고:


## 절대 규칙

1. **PART B(이 파일 하단의 스킬 전문)를 반드시 숙지하고 적용한다.** 8원칙·양식 디테일·직무별 패턴이 모두 거기 있음.
2. **history.json을 반드시 먼저 읽고 시작**한다. 안 읽고 무작위로 뽑으면 중복 발생.
3. **used_combos에 있는 (직무|연차|상황) 조합은 절대 다시 뽑지 않는다.**
4. **remaining_in_cycle에 없는 직무는 이번 사이클에 절대 뽑지 않는다.** (사이클 균등 분배 원칙)
5. **가상 인물의 회사명에 실제 기업명을 사용하지 않는다.** (○○패션, △△제약 등 마스킹)
6. **양식 A와 B는 같은 가상 인물·같은 경력의 다른 표현**이지, 다른 사람이 아니다.
7. **PART B의 8가지 원칙과 연차별 현실적 성과 수준을 반드시 준수**한다. 1년차에게 "팀 전체 전략 수립" 같은 과장 금지.
8. **history.json 업데이트는 docx 6개 생성 완료 후 한 번에**. 중간에 실패하면 history는 업데이트하지 않는다.

# PART B: 경력기술서 작성 스킬 전문

(아래는 457deep의 경력기술서 작성 스킬 본문이다. 위 STEP 8의 docx 생성 시 이 내용을 모두 적용한다.)

---

# 사기업 경력기술서 작성 스킬

## 개요

457deep의 경력기술서 8가지 작성 원칙(최채용솔로가정부)과 20가지 합격 예시, 실제 합격 경력기술서 사례를 기반으로, 직무·연차·이직 상황에 맞춤화된 경력기술서를 작성하는 스킬이다.

## 입력

1. **직무** (필수) — 지원하려는 직무명 (예: 영업, 마케팅, 재무회계, 생산관리, MD, IT 등)
2. **연차/직급** (필수) — 현재 연차 및 직급
3. **이전 경력 정보** (필수) — 회사명, 부서, 근무기간, 주요 업무 내용
4. **주요 성과** (권장) — 정량/정성 성과. 없으면 업무 내용으로 추론하여 작성
5. **특수 상황** (선택) — 직무 전환, 업계 변경, 경력 단절, 인턴 경험만 있는 경우 등
6. **채용 공고/JD** (선택) — 지원 기업의 채용 공고가 있으면 JD에 맞춤화
7. **분량 선호** (선택) — 짧은 버전 / 중간 버전 / 긴 버전

## 출력

- docx 형태의 완성된 경력기술서
- 개조 형식 기본 (필요 시 개조+서술 혼용)
- 직무·연차·상황에 맞는 양식 자동 선택

---

## 1. 작성 프로세스

```
STEP A: 입력 정보 분석 — 직무 특성, 연차, 이직 상황 파악
STEP B: 양식 결정 — 업무 로직(루틴/프로젝트)에 따라 양식 선택
STEP C: 업무 내용 정리 — 과업 단위로 세분화, 대분류/소분류 카테고리화
STEP D: 성과 기술 — HOW(행동) + Result(성과) 구조로 정량/정성 성과 작성
STEP E: 8가지 원칙 점검 — 최채용솔로가정부 원칙 전체 적용 확인
STEP F: 가독성 최종 점검 및 docx 출력
```

---

## 2. 경력기술서 8가지 핵심 원칙 (최채용솔로가정부)

### 원칙 1: 최(최신) — 최신 경력부터 작성

- 시간순이 아니라 **최근 경력부터 역순**으로 기술
- 최근 경력이 현재 역량을 가장 잘 드러냄
- 지원 직무와 관련성 높은 경력을 먼저 배치
- 경력 단절이 있는 경우 최신 경력 우선이 특히 유리

### 원칙 2: 채(채용공고) — JD에 맞추어 작성

**과업 단위로 세분화하여 JD와 매칭:**
- "인사 업무 전반 담당" ❌ → "채용 전략 기획 및 프로세스 개선 / 면접 운영 및 평가 시스템 고도화" ✅
- 채용 공고의 주요 업무, 자격 요건, 우대 사항을 파악한 뒤 본인 과업과 매칭
- 채용 공고에 언급된 프로그램, tool, 설비 활용 경험 반드시 어필

**작성하면 안 되는 과업 (불필요한 업무 제거):**
- 일상적인 소셜 미디어 포스팅, 단순 데이터 입력, 사내 게시판 관리
- 단순 견적서 작성, 실험 기구 세척, 생산 현장 청소 등 단순 보조 업무

**특수 상황 대응:**
- 유사 직무 전환 시: 전환 직무와 매칭되는 과업을 최대한 발굴하여 기술
- 완전히 다른 직무 이직 시: 지원 직무 관점으로 경력 재각색 및 매칭
- 특별히 지원 회사가 정해지지 않은 경우: 해당 직무/업계 채용 공고 10개 이상 확인 후 공통 업무 파악

### 원칙 3: 용(용어) — 직무/업계 맞춤 용어 사용

- 채용 공고의 키워드와 용어를 자연스럽게 녹여내기
- 해당 분야/직무에서만 통용되는 KPI, 실무 용어 활용
- 이전 직장/업계 용어가 지원 업계와 맞지 않으면 변경
- **직무 특성에 맞는 용어를 사용해야 "이 사람은 실무를 아는 사람"으로 인식됨**

**직무별 용어 예시:**
- 마케팅: ROAS, CVR, CPA, 리타겟팅, 퍼널, CAC, 오가닉, 인게이지먼트
- 영업: 파이프라인, 리드, 아웃바운드, CRM, 수주율, 업셀링, 클로징
- 인사: 역량 기반 면접, 조직문화, 이직률, 온보딩, 리텐션, ATS
- 생산: 리드타임, OEE, SPC, Lean, Six Sigma, 불량률, 가동률
- IT: Agile, CI/CD, DevOps, 코드 리뷰, 테스트 커버리지, 스프린트
- MD: SKU, ASP, 바잉, 시즌 오더, 기획전, 상품 회전율, 소싱, ST%
- 제약영업: 디테일링, KOL, SFE, IMS/유비스트, 약제위원회, 코드인, Veeva CLM, 처방 점유율
- 재무/회계: FI/CO, 결산, 원천세, 재무제표, 감가상각, 대손충당, 관리회계
- 물류/SCM: WMS, 3PL, 재고 회전율, 납기 준수율, 리드타임, 안전 재고

**용어 활용 수준:**
- 업무 내용 기술 시: 해당 직무에서 일상적으로 사용하는 실무 용어로 표현
- 성과 기술 시: KPI 용어를 사용하여 성과를 측정 가능한 형태로 표현
- 과도한 용어 남발 금지: 문맥에 자연스럽게 녹아들어야 함

### 원칙 4: 솔(솔직) — 솔직하게 작성

- 과장 금지: 레퍼런스 체크 검증 가능성 높음
- 면접 일관성: 경력기술서 기반으로 면접 질문이 나옴
- 자기 객관화: 강점/약점 파악의 기회
- 직책/역할 과장, 성과 부풀리기, 근무기간 늘리기, 기술 능력 부풀리기 절대 금지

### 원칙 5: 로(로직) — 업무 로직에 맞춰 양식 결정

**양식 선택 기준:**

| 업무 로직 | 양식 | 해당 직무 예시 |
|-----------|------|----------------|
| 루틴 업무 중심 | 업무 단위로 기술 | 행정, 회계/재무, 생산/품질, 고객서비스, 법무, 물류, 구매, 영업관리, MD |
| 프로젝트 중심 | 프로젝트 단위로 기술 | R&D, 마케팅 캠페인, IT/PM, 건설, 이벤트 기획, M&A, 신사업 개발 |
| 혼합형 | 업무 단위 기술 + 특정 프로젝트 성과 강조 | 대부분의 직무 |

**루틴 업무의 특징:**
- 업무 책임 범위(scope)가 핵심 평가 요소
- 정해진 절차/프로세스가 명확
- 업무가 연속적 → 근무 기간 자체가 역량 지표
- 업무 지식/전문성을 강조해서 기술

**프로젝트 업무의 특징:**
- 시작과 끝을 담당, 목표 지향적
- 문제 해결력이 핵심 평가 포인트
- 업무가 비연속적/분절적

**주의: 지원 직무의 로직에 맞추기**
- 루틴 직무인데 프로젝트 단위로 성과 과장 → 부자연스러움
- OKR 중시 기업 지원 시 → 루틴 업무도 목표/성과 위주로 전환 가능

### 원칙 6: 가(가독성) — 가독성 있게 구성

- 간결하고 명확한 문장: 한 문장에 1~2개 정보만
- 대분류/소분류로 나누어 문장 간결하게 다듬기
- 일관된 포맷과 구조 유지 (글씨체, 크기, 줄간격)
- 핵심 정보를 문단 맨 앞에 배치
- 적절한 여백과 구분선으로 그룹핑
- 풀어쓰기 대신 전문용어/KPI 용어로 대체하여 간결하게

**Before/After 예시:**
- Before: "고객사 방문 뒤, 요구 사항을 반영하여 프로젝트 수행"
- After: "고객 요건 분석 및 단계별 프로젝트 기획 (WBS 작성)"

### 원칙 7: 정(정량/정성) — 정량·정성 성과 강조

**업무 내용 정량화 방법:**
- 업무 수행 범위: "전국 5개 지역, 100여 개 매장 관리"
- 대상 수: "월 평균 200명의 고객 상담"
- 빈도 수: "분기별 성과 리뷰", "일 평균 5~10건 VOC 분석"
- 규모: "연간 예산 10억 원 규모 마케팅 집행"

**업무 내용 정성화 방법 (난이도/중요성/전문성):**
- 난이도: "복잡한 대규모 시스템 성능 저하 이슈 해결"
- 중요성: "회사의 DX 전략 구현하는 핵심 시스템 구축 프로젝트 리딩"
- 전문성: "글로벌 선진 기업 품질관리 프로세스 벤치마킹 및 도입"

**비매출 직무의 성과 정량화 — QCD 관점:**
- Quality(품질): 불량률 감소, 오류율 감소, 합격률 개선
- Cost(비용): 비용 절감, 에너지 절약, 인건비 절감
- Delivery(납기): 리드타임 단축, 납품률 향상, 처리 시간 단축

**정량/정성 비중 가이드:**
- 정량 성과가 중요한 직무 (영업, 생산, 마케팅): 정량 70~80% + 정성 10~20%
- 정량 성과가 어려운 직무 (인사, 디자인, 총무): 정량 40~50% + 정성 40~50%

**정성적 성과가 적합한 경우:**
- 인턴/저연차로 기여도가 미미한 경우: "데이터 신뢰성에 기여", "업무 효율성 제고에 기여"
- 본인 역량/태도 성장: "데이터 분석 역량 향상", "전산 프로그램 역량 향상"
- 정성적 성과의 전문성이 너무 뛰어나 정량 불필요: "그래핀 기반 복합소재 개발"
- 퇴사하여 이후 정량 성과 확인 불가: "신규 시장 진출에 기여"

### 원칙 8: 부(부합성) — 행동의 부합성 고려

**핵심: HOW(행동) + Result(성과) 구조**
- "불필요한 예산 15% 절감" ❌
- "출장여비 기준 현실화로 불필요한 예산 15% 절감" ✅

**3가지 부합성 점검:**

1) **직무 행동 역량 부합**: 행동이 해당 직무에서 요구하는 역량과 일치해야 함
   - MD가 "50% 할인 기획전으로 매출 증대" ❌ → "할인/정상가 최적 믹스 전략으로 매출 12% 증대, ASP 3% 향상" ✅

2) **구체성 확보**: 추상적 행동 → 구체적 행동으로 전환
   - "고객 니즈 분석으로 매출 증가" ❌ → "고객 업종 분석으로 잠재 니즈 파악, 통합 서비스 제안으로 신규 수주 00곳 달성" ✅

3) **연차/직급 부합**: 연차에 맞는 수준의 행동과 전략
   - 대리급에서 "꼼꼼한 업무 수행으로 장부 정확성 100%" ❌ (태도적 접근)
   - 대리급에서 "비용 절감 기회를 분석하여 연간 예산 5% 절감" ✅ (전략적 접근)

**연차별 기대 행동 수준:**
- 신입/인턴 (0~1년): 기본 업무 습득, 태도적 접근 허용
- 사원 (2~3년): 독립적 업무 수행, 소규모 문제 해결
- 대리 (4~5년): 문제 근본 원인 파악, 데이터 기반 분석, 프로세스 개선
- 과장 (6~9년): 팀 리더십, 전략 수립, 부서 간 조정
- 차장 이상: 핵심 의사결정, 전략 수립, 대외 협력 주도

**연차별 현실적 성과 수준 가이드 (필수 준수):**

경력기술서의 업무 내용과 성과는 해당 연차가 현실적으로 수행할 수 있는 수준이어야 한다. 과장된 성과는 면접에서 검증되면 오히려 역효과가 발생한다.

| 연차 | 현실적 수준 | 과장된 수준 (금지) |
|------|-------------|---------------------|
| 1년차 | 선배 동행 학습 후 독립 영업, 신규 거래처 5~10곳 개척, 목표 100~110% 달성 | 팀 전체 전략 수립, KOL 직접 미팅, 목표 150% 달성 |
| 2~3년차 | 독립 담당 영역 운영, 선배 서포트로 프로젝트 참여, 목표 105~115% 달성, 특정 프로모션에서 KPI 초과 1~2건 | 약제위원회 등재 전담 3건, 전사 전략 수립, 목표 매년 130% 달성 |
| 4~5년차 | 프로세스 개선 주도, 후배 지도, 핵심 프로젝트 리딩, 목표 110~120% 달성 | 사업부 전체 전략 총괄, 임원급 의사결정 참여 |

**임팩트 성과 배치 가이드:**
- 대부분의 성과는 해당 연차의 현실적 수준으로 기술
- 단, 1~2건은 KPI를 크게 초과 달성한 임팩트 성과를 배치 (특정 프로모션, 특정 기간 한정)
- 임팩트 성과도 "어떻게 달성했는지" HOW를 반드시 구체적으로 기술해야 설득력이 있음

**트렌드/혁신성 강조:**
- AI 활용, 디지털 전환, OKR 도입, ESG 보고 체계 등
- 직무/업계에서 트렌디한 과업을 수행했다면 반드시 기술

## 3. 경력기술서 필수 기재 항목

```
1. 회사명 또는 업계명 (예: 패션업계, 식품업계, IT업계, 건설업계, 편의점업계, 제약업계 등. 파견직은 소속 회사/업계로)
2. 직급 및 역할
3. 회사 설명 (양식에서 요구 시)
4. 구체적 근무 기간 (20xx.x 또는 20xx.x.x)
5. 구체적 업무 내용 (과업 단위)
6. 정량/정성 성과
7. 적용한 지식·기술적 역량 (선택, IT/기술 직군은 필수)
```

## 4. 성과 기술 공식

### 기본 공식: HOW + Result

```
[행동/전략/노력] + [정량 또는 정성 성과]
```

예시:
- "생산 일정 최적화 및 병목 공정 개선을 통해 생산 리드타임 20% 단축"
- "공급업체 10곳→17곳 다변화로 원자재 구매 비용 10% 절감"
- "맞춤화된 직원 역량 강화 교육 프로그램 기획·운영으로 이직률 15% 감소"

### HOW 구체화 규칙 (필수)

**성과만 나열하지 말고, 반드시 "어떻게 달성했는지"를 하위 항목으로 구체 기술한다.**

```
• 성과 요약 (볼드)
  - HOW 1: 구체적 행동/전략
  - HOW 2: 구체적 행동/전략
  - HOW 3: 구체적 행동/전략
```

**HOW 작성 시 반드시 포함해야 할 요소:**
- 어떤 데이터/정보를 활용했는지 (IMS 데이터, 매출 현황, 경쟁사 분석 등)
- 어떤 행동을 취했는지 (방문 빈도 조정, 자료 자체 제작, 리스트 작성 등)
- 어떤 결과로 이어졌는지 (처방 연결, 관계 형성, 접점 확대 등)

**HOW 작성 시 금지 사항:**
- → 화살표 기호 사용 금지. 반드시 - (대시) 또는 · (가운데점) 기호 사용
- 당연한 행동은 HOW로 쓰지 않음 (예: "성실하게 방문했다", "매뉴얼을 준수했다")
- 누구나 하는 기본 업무를 성과로 포장하지 않음

**HOW 구체화 예시:**

나쁜 예:
```
• 신규 거래처 8곳 확보
  - 꾸준한 방문으로 관계를 구축함
```

좋은 예:
```
• 신규 거래처 8곳 개척
  - 지역구 의사회 및 대학교 동문 모임 등 의료진 커뮤니티를 파악하여 제품 설명회 참석 기회 확보
  - 설명회를 통해 접점을 만든 의료진에게 후속 방문 및 샘플 전달로 처방 연결
  - 신제품 코드인을 활용하여 미처방 의원의 초회 처방 진입 장벽을 낮추고 정기 처방으로 전환
```

### 임팩트 성과 필수 포함 규칙

**경력기술서에 반드시 1~2건의 임팩트 있는 성과(KPI 초과 달성 사례)를 포함한다.**

임팩트 성과란:
- 목표 대비 크게 초과 달성한 사례 (120% 이상)
- 특정 프로모션/캠페인에서 두드러진 성과를 낸 사례
- 본인만의 차별화된 전략으로 성과를 만든 사례

임팩트 성과 기술 방법:
```
• 20xx년 O분기 신제품 개척 프로모션 기간 내 개척율 140% 달성
  - 신규 개원 의원 정보를 사전에 수집하여 오픈 초기 방문 공략 리스트 작성
  - 개원 축하 감성 이벤트(화환·개원 축하 서신)로 초기 관계 형성
  - 판촉물(처방 가이드 포켓북, 진료실 비품)을 활용한 방문 빈도 확대로 의료진 접점 극대화
  - 개척 대상 의원별 환자군 특성을 사전 분석하여 맞춤형 디테일링 포인트 준비
```

**주의: 임팩트 성과도 해당 연차에서 현실적으로 달성 가능한 수준이어야 함**

### 당연한 성과 제거 규칙

**아래에 해당하는 내용은 성과로 기술하지 않는다:**
- 입사 후 독립 영업 전환 (모든 신입이 거치는 과정)
- 매뉴얼/절차를 준수한 것 (기본 업무 수행)
- 교육을 이수한 것 (자기 성장이지 성과가 아님)
- 일정을 준수한 것 (당연한 의무)
- 선배에게 피드백을 받은 것 (학습 과정이지 성과가 아님)

### STAR 기법 활용 (경력기술서 요약형)

```
주요 업무 = Task (역할, 업무 내용)
주요 성과 = Action(행동) + Result(성과)
```

### 성과 기술 변형

**AR 방식 (기본):**
```
주요 업무: 고객 데이터 분석 및 세그먼테이션
주요 성과: 5개 핵심 고객 페르소나 개발로 신규 고객 유입 45% 증가
```

**TAR 방식 (업무 내용 보완):**
```
(마케팅 전략) 5개 핵심 고객 페르소나 개발로 신규 고객 유입 45% 증가
 TASK          ACTION                       RESULT
```

**성과 강조형 (성과→행동 역순):**
```
주요 성과: 이메일 마케팅 오픈율 15%→28% 상승
- 오픈율 부진 포인트 재분석
- 세그먼트별 맞춤형 이메일 캠페인 전략 수립
```

## 8-1. 직무별 성과 전략 패턴 (HOW 구체화 레퍼런스)

**경력기술서의 성과는 "무엇을 달성했는가"뿐 아니라 "어떤 직무적 사고와 전략으로 달성했는가"가 핵심이다.**
아래는 직무별로 자주 활용되는 성과 전략 패턴이다. 해당 직무의 경력기술서를 작성할 때 이 패턴을 참고하여 HOW를 구체화한다.

### 제약영업 (MR)

**핵심 전략 패턴:**
- **데이터 기반 틈새 전략**: IMS/유비스트 처방 데이터를 품목·용량·진료과 단위로 세분화 분석하여 자사 점유율이 낮은 틈새 영역을 도출, 해당 영역에 집중 디테일링
- **경쟁사 이슈 기반 전환 영업**: 경쟁사 품목의 공급 불안정·부작용 이슈·약가 변동 등을 현장에서 조기 감지하여 자사 제품 대체 제안으로 전환 처방 확보
- **거래처별 차별화 전략**: 동일 품목이라도 거래처별 상이한 접근 (A: 증량 확대 / B: 미개척 품목 신규 진입 / C: 경쟁사 전환)
- **커뮤니티 활용 개척**: 의사회·대학 동문 모임 등 의료진 커뮤니티를 파악하여 제품 설명회 기회 확보, 후속 방문으로 처방 연결
- **감성 이벤트 기반 관계 형성**: 신규 개원 축하 이벤트, 판촉물 활용 등으로 초기 라포 형성 후 처방 연결
- **신제품 코드인 전략**: 코드인을 활용하여 미처방 거래처의 초회 처방 진입 장벽을 낮추고 정기 처방으로 전환

**성과 기술 예시:**
```
• 20xx년 O분기 신제품 개척 프로모션 기간 내 개척율 140% 달성
  - 신규 개원 의원 정보를 사전에 수집하여 오픈 초기 방문 공략 리스트 작성
  - 개원 축하 감성 이벤트(화환·개원 축하 서신)로 초기 관계 형성
  - 판촉물(처방 가이드 포켓북, 진료실 비품)을 활용한 방문 빈도 확대로 의료진 접점 극대화
  - 개척 대상 의원별 환자군 특성을 사전 분석하여 맞춤형 디테일링 포인트 준비
```

### 영업 (일반/B2B)

**핵심 전략 패턴:**
- **고객 업종 분석 기반 제안**: 고객사의 산업 특성·사업 구조를 분석하여 맞춤형 솔루션 제안으로 수주 확보
- **CRM 데이터 기반 이탈 방지**: 계약 만료 시점·구매 주기를 트레킹하여 선제적 리텐션 영업 실행
- **경쟁사 약점 공략**: 경쟁사 제품/서비스의 불만 포인트를 현장에서 수집, 자사 강점을 대비시킨 제안서 작성
- **크로스셀/업셀 전략**: 기존 거래처의 미활용 제품·서비스를 분석하여 추가 매출 확대
- **VOC 기반 피드백 루프**: 고객 현장 의견을 체계적으로 수집·전달하여 제품/서비스 개선에 기여

**성과 기술 예시:**
```
• 기존 고객사 크로스셀 전략으로 고객당 평균 거래액 15% 증대
  - 기존 거래 품목 외 미활용 제품군을 고객사별로 분석하여 추가 제안 리스트 작성
  - 고객사 현장 방문 시 실사용 환경을 파악하여 실질적으로 필요한 제품을 맞춤 제안
  - 시범 도입 후 사용 후기를 수집하여 본계약 전환율 향상
```

### 마케팅

**핵심 전략 패턴:**
- **타겟 세분화 전략**: 고객 데이터를 세그먼트별로 분석하여 페르소나 기반 캠페인 기획
- **A/B 테스트 기반 최적화**: 소재·채널·타이밍별 A/B 테스트를 통해 전환율·ROAS 개선
- **콘텐츠 기반 유입 전략**: SEO 최적화, 트렌드 연계 콘텐츠 기획으로 오가닉 유입 확대
- **프로모션 ROI 분석**: 프로모션별 비용 대비 성과를 분석하여 고효율 채널에 예산 재배분
- **경쟁사 벤치마킹**: 경쟁 브랜드의 캠페인·프로모션을 모니터링하여 차별화 포인트 발굴

### MD / 바잉

**핵심 전략 패턴:**
- **판매 데이터 기반 카테고리 믹스 최적화**: SKU별 ST%, ASP, 재고 회전율을 분석하여 시즌 바잉 구성 조정
- **할인/정상가 믹스 전략**: 할인 상품과 정상가 상품의 최적 비율을 설계하여 매출과 수익성 동시 확보
- **PB/기획 상품 개발**: 시장 니즈 분석을 통해 자체 기획 상품을 제안·개발하여 마진율 개선
- **부진 상품 조기 대응**: 판매 추이 모니터링으로 부진 상품을 조기 발견, 프로모션·진열 변경으로 소진 유도
- **트렌드 선제 반영**: 경쟁사·해외 트렌드를 분석하여 시즌 바잉에 선제 반영

### 생산관리

**핵심 전략 패턴:**
- **병목 공정 분석·개선**: 공정별 데이터를 분석하여 병목 지점을 도출, 작업 순서·설비 배치 재조정으로 리드타임 단축
- **예방 정비 체계 구축**: 설비 고장 이력 데이터를 분석하여 예방 정비 주기를 최적화, 돌발 정지 감소
- **불량 원인 분석**: 불량 유형별 파레토 분석 후 상위 원인에 대한 개선 조치 실행으로 불량률 감소
- **작업 표준화**: 작업자별 편차를 분석하여 표준 작업 지침서를 작성, 품질 균일화 달성

### 재무/회계

**핵심 전략 패턴:**
- **비용 구조 분석**: 계정 항목별 시계열 분석으로 비정상적 비용 증가 패턴을 발견하여 개선 제안
- **프로세스 자동화**: 반복 업무(전표 처리, 데이터 검증)를 엑셀 매크로·RPA로 자동화하여 업무 시간 절감
- **예산 실적 차이 분석**: 부서별 예산 집행 현황을 분석하여 과집행·미집행 원인을 파악, 예산 운영 효율화

### 인사(HR)

**핵심 전략 패턴:**
- **채용 퍼널 분석**: 단계별(서류→면접→합격) 전환율을 분석하여 이탈이 많은 단계에 개선 조치 실행
- **만족도 데이터 기반 조직 진단**: 설문 결과를 부서별·항목별로 분석하여 핵심 불만 요인을 도출, 맞춤 개선 과제 실행
- **리텐션 전략**: 이직 사유 데이터를 분석하여 핵심 인재 이탈 방지 프로그램 기획

### IT/개발

**핵심 전략 패턴:**
- **성능 프로파일링 기반 최적화**: 성능 측정 도구로 병목 지점을 식별하여 타겟 최적화 실행
- **자동화 파이프라인 구축**: 반복 작업(테스트, 빌드, 배포)을 자동화하여 개발 사이클 단축
- **사용자 피드백 기반 개선**: 사용자 행동 데이터·VOC를 분석하여 UX 개선 우선순위 설정

### 물류/SCM

**핵심 전략 패턴:**
- **수요 예측 모델 활용**: 과거 판매 데이터를 기반으로 수요를 예측하여 적정 재고 수준 유지, 과잉/결품 최소화
- **동선/루트 최적화**: 배송 루트를 분석하여 동선을 재설계, 운송 비용 및 리드타임 절감
- **공급업체 다변화**: 단일 공급 리스크를 분석하여 대체 공급처를 확보, 공급망 안정성 향상

### 홍보/PR

**핵심 전략 패턴:**
- **매체별 맞춤 스토리텔링**: 매체 특성(일간지, 경제지, 온라인)에 맞춘 앵글 차별화로 기사화율 향상
- **위기 조기 감지 체계**: 일상적 모니터링으로 부정 이슈를 조기 발견, 초기 대응 시간 단축
- **기업 활동 연계 콘텐츠**: 기업의 실제 활동(ESG, 사회공헌 등)을 스토리텔링 소재로 기획하여 긍정 노출 확대

---

### 직무별 전략 패턴 활용 규칙

1. **해당 직무의 전략 패턴을 참고하여 HOW를 구체화한다** — 성과만 나열하지 말고, 위 패턴 중 적합한 전략을 성과의 HOW 하위 항목에 녹여낸다
2. **거래처/고객별 차별화 전략을 반드시 포함한다** — "모든 거래처에 동일하게 했다"는 전략이 아니다. 거래처/고객별로 다르게 접근했음을 보여줘야 역량이 드러남
3. **데이터 분석 → 원인 파악 → 전략 수립 → 실행 → 결과**의 흐름이 보여야 한다
4. **경쟁사 동향 활용 전략을 포함한다** — 경쟁사 이슈를 파악하여 자사 제품/서비스로 전환·대체한 경험은 높은 평가를 받는 전략

## 10. docx 생성 규칙

1. **파일명**: `경력기술서_[이름]_[직무].docx` (이름 모르면 `경력기술서_[직무].docx`)
2. **성명**: 사용자가 이름을 제공하지 않으면 **OOO**으로 표기
3. **학력**: 사용자가 학교명을 제공하지 않으면 **OO대학교**로 표기
4. **폰트**: 맑은 고딕 또는 나눔고딕
5. **본문 크기**: 10~11pt
6. **제목/소제목**: 볼드, 11~13pt
7. **줄간격**: 1.2~1.5
8. **여백**: 상하좌우 2cm
9. **구조**: 개조 형식 기본. 필요 시 개조+서술 혼용
10. **기본 정보(성명, 경력 요약 등)**: 테이블(칸) 없이 텍스트 형식으로 작성
11. **기호 표기 규칙**:
    - 성과 요약: • (볼드)
    - 성과 HOW 하위 항목: - (대시) 또는 · (가운데점)
    - → 화살표 기호 사용 금지
    - 업무 내용 항목: - (대시)
---

## 11. 최종 점검 체크리스트

작성 완료 후 반드시 아래 항목을 점검:

- [ ] 최신 경력부터 역순 배치했는가?
- [ ] JD/지원 직무에 맞는 과업으로 매칭했는가?
- [ ] 직무/업계 맞춤 용어를 활용했는가? (해당 직무 실무자가 읽었을 때 자연스러운가?)
- [ ] 과장/허위 내용이 없는가?
- [ ] 업무 로직(루틴/프로젝트)에 맞는 양식인가?
- [ ] 가독성이 확보되었는가? (간결한 문장, 일관된 포맷)
- [ ] 정량/정성 성과가 적절히 포함되었는가?
- [ ] HOW+Result 구조로 성과를 기술했는가? (성과마다 구체적 행동 하위 항목 포함)
- [ ] 행동이 직무 역량·연차와 부합하는가? (해당 연차에서 현실적으로 수행 가능한 수준인가?)
- [ ] 불필요한 업무(단순 보조, 본인 직급 이하 업무)를 제거했는가?
- [ ] 당연한 성과(독립 전환, 매뉴얼 준수, 교육 이수 등)를 제거했는가?
- [ ] 임팩트 있는 성과(KPI 초과 달성)가 1~2건 포함되어 있는가?
- [ ] 기호 표기가 통일되어 있는가? (→ 화살표 미사용, - · • 기호 통일)
